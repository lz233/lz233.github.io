reMarkable 2 Hack 不完全指北

发布于 2021-07-12 reMarkable Linux 

进行下列操作前务必明确你在干什么。任何操作都请记得备份，否则后果自负。

reMarkable 2 是一款用于笔记与批注的水墨屏设备，因其的高颜值、低延迟和高开放性等优点，而深受我的喜爱。

rM 在国内的知名度很低，相关中文资料很少，故有此文。

SSH

rM 默认开启了 SSH 端口，转到 Settings -> Help -> Copyrights and licenses -> GPLv3 Compliance 查看可用的 IP 地址和密匙，并用如下命令连接设备：

ssh root@IP

先来借助 neofetch 查看一下 rM2 的技术细节吧！

中文字体

rM 在其市场定位的选择中并没有将中国大陆包含在内，故无自带中文。

Linux 的字体存放在 /usr/share/fonts，将字体通过 SCP 移入即可：

scp your_font(s) root@IP:/usr/share/fonts/ttf

个人比较常用的字体是未来荧黑和更纱黑体，下载完毕后须删除等宽体等变体，因为 rM 不会让你选择字体，.otf 文件应放在 /usr/share/fonts/opentype。

如提示无储存空间，可通过建立符号链接解决，示例如下：

mkdir ~/fonts //on rM
scp ./glow root@10.11.99.1:~/fonts //on PC
ln -s ~/fonts /usr/share/fonts/opentype //on rM

完成操作后重启生效。

休眠壁纸

有关开机、关机、休眠状态下的图片都在 /usr/share/remarkable 下，重启后生效，怎么处理都随你啦！

Opkg

多亏了 rM 的高度开放性，社区已经提供了一键安装脚本，使用如下脚本完成 Opkg 的安装，然后，你就可以使用来自 Entware 和 Toltec 的软件包。

wget -O - http://raw.githubusercontent.com/Evidlo/remarkable_entware/master/install.sh | sh

完成后使用 nano 在 /opt/etc/opkg.conf 添加三行配置，方便后续操作。

arch arm 100
arch rm2 100 //rm2
arch rm1 100 //rm1
arch rmall 100

rm2fb

rM2 需要使用 rm2fb 驱动帧缓冲区。当前阶段（rM version>=2.6），务必使用 Toltec Testing 源中的软件包，否则可能无法正常工作。

Testing 源不提供直接索引，将 .ipk 文件下载后手动安装：

wget https://toltec-dev.org/testing/rm2/display_1.0.1-6_rm2.ipk //replace your version
wget https://toltec-dev.org/testing/rm2/rm2fb-client_1.0.1-6_rm2.ipk //replace your version
opkg install *.ipk

remux

remux 是为 rM 设计的多任务应用启动器。当前依然建议安装 Testing 源的软件包，安装重启后在屏幕左侧或右侧向上划（幅度大点）即可打开启动器。

修改 ~/.config/remux/remux.conf 可自定义唤起方式，例如，下方配置代表三指点击（有很大的误触几率）唤起启动器。

launch_gesture=gesture=tap;fingers=3

Java

目前可以从 AdoptOpenJDK 获取到 arm 架构的 jdk 16 和 jre 16。下载解压后在 ~/.bashrc 最后一行配置 PATH 变量即可。

PATH=/opt/bin:/opt/sbin:$PATH:/home/root/your_java_dir/bin

FileZilla

FileZilla 是一款通过 FTP/SFTP 传输文件的工具，你可以使用它代替上文提到的 SCP，使用图形化界面进行操作。

其他

上文提到的 Toltec 源是专门为 rM 提供的，你还可以尝试源中的其他软件。这里说一些需要特别注意的地方：

  • calculator 推荐使用 Testing 源
  • fingerterm 推荐使用 Testing 源
  • wikipedia 推荐使用 Testing 源
  • remarkable-hacks 很大概率上会造成 xochitl 极其不稳定（至少在我这是这样），不推荐使用。
  • 使用第三方启动器后，建议取消 PIN，因为这可能会造成 rM 失控。
  • 终端推荐使用 yaft。这比 fingerterm 快且易用。
  • 一份更加丰富的社区指南已经为你呈现：awesome-reMarkable。
